# Comparación de Programación Tradicional y Programación Orientada a Objetos en Python

## Descripción del proyecto
Este proyecto tiene como objetivo comparar la Programación Tradicional y la Programación Orientada a Objetos (POO) mediante el desarrollo de un programa en Python que calcula el promedio semanal del clima a partir de temperaturas ingresadas por el usuario.

El mismo problema se resuelve utilizando dos enfoques diferentes con el fin de analizar sus características, ventajas y diferencias.

---

## Programación Tradicional
En este enfoque, el programa se desarrolla utilizando funciones independientes para realizar cada tarea, como el ingreso de las temperaturas diarias y el cálculo del promedio semanal. Los datos se manejan mediante listas y se siguen estructuras secuenciales.

Este tipo de programación es sencilla y fácil de entender, siendo adecuada para programas pequeños y con poca complejidad.

---

## Programación Orientada a Objetos (POO)
En el enfoque de Programación Orientada a Objetos, se utiliza una clase que representa el clima semanal. Esta clase encapsula los datos (temperaturas) y los métodos necesarios para ingresar la información y calcular el promedio.

La POO permite una mejor organización del código, facilita la reutilización y mejora el mantenimiento del programa, especialmente en proyectos más grandes.

---
